## Security & Compliance Principles

### Data Access Hierarchy:
1. **Own Data**: All roles can access their own personal information
2. **Assigned Data**: Access to students, courses, or leads assigned to them
3. **Team Data**: Supervisors can access their team's aggregated data
4. **Organizational Data**: Admins have access to all organizational data

### Iranian Market Compliance:
- **Shetab Payment Integration**: Only Accountant and Admin roles
- **Kavenegar SMS Access**: Call Center, Teachers, and Admin roles
- **Local Data Storage**: All roles must work with locally stored data
- **Persian/Arabic Support**: All interfaces support RTL languages

### Audit & Logging:
- All actions are logged with user attribution
- Financial transactions require dual approval (Admin + Accountant)
- Grade changes require supervisor notification
- System changes require admin approval

### Progressive Disclosure:
- Basic users see simplified interfaces
- Advanced users get comprehensive dashboards
- Emergency overrides available for critical situations
- Role-specific onboarding and help systems
